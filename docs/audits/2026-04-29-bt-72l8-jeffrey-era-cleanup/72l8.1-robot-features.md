## Methodology

Note on flag form: AGENTS.md and bt-72l8.1's framing reference `--robot-*` flags. Those have been gone for a while — the current surface is cobra subcommands under `bt robot <name>`. The feature classification holds either way; this audit uses the live form.

Discovery:
- `bt robot --help` enumeration → 30 top-level subcommands.
- Nested help (`bt robot sprint|correlation|baseline|labels|files`) → 13 nested subcommands.

Invocation: each subcommand run from `C:\Users\sms\System\tools\bt` against bt's own beads, with `BT_NO_BROWSER=1 BT_TEST_MODE=1` and `timeout 25`. For arg-required commands, ran with valid bt IDs (`bt-53du`, `cmd/bt/main.go`, `pkg/ui`, `bt robot search --query graph`, etc.).

Source-side cross-reference for GHOSTs: traced every "no beads file found" error to its origin. All paths land at `pkg/correlation/correlator.go:365` — `fmt.Errorf("no beads file found in %s/.beads/", repoPath)`. The correlator requires `.beads/<project>.jsonl`, which the Dolt-only bd backend does NOT produce. Confirmed empty: `ls .beads/` shows `dolt/`, `config.yaml`, `correlation_feedback.jsonl`, etc., but no `<project>.jsonl`. This matches AGENTS.md's "Stale-assumption checklist" and bt-08sh's tracking of the correlator Dolt migration.

## GHOST features (missing producers)

Two distinct producer-missing buckets:

### Bucket 1: Correlator JSONL (8 GHOSTs)
Subcommands: `orphans`, `history`, `causality`, `related`, `impact`, `files hotspots`, `files beads`, `files relations`.

What it expects: `.beads/<project>.jsonl` written by legacy bd JSONL backend.
Why broken: Beads is Dolt-only since v1.0.1 (March 2026). No producer writes the JSONL. Error surfaces from `pkg/correlation/correlator.go:365`.
Already tracked: **bt-08sh** (P2) — "bt correlator: migrate from `.beads/<project>.jsonl` + git-diff witness to `dolt_log`/`dolt_history_issues`". This audit confirms scope: 8 robot subcommands all wedge on the same producer gap.

### Bucket 2: Sprint feature (3 GHOSTs)
Subcommands: `sprint list`, `sprint show`, `burndown`.

What it expects: `.beads/sprints.jsonl` written by some producer.
Why broken: Sprints aren't a beads concept upstream (Dolt schema has no sprints table; bd has no sprint subcommand). Per AGENTS.md, "Full sprint CRUD requires bd CLI changes" — never landed, beads is now Dolt-only.
Already tracked: **bt-z5jj** (P3) — rebuild against Dolt or retire. **bt-3suf** (P3) child — explicit retirement track.
This audit confirms three robot subcommands are functionally hollow under this gap (not just one).

## PARTIAL features

- **`docs`** — emits structured docs keyed on legacy `--robot-*` flag names. Field/param info still useful for agents, but flag references no longer match the cobra surface. Cosmetic but misleading to consumers.
- **`pairs`** — works under `--global` but quality is low: ~5x false positive rate per MEMORY.md (suffix collisions dominate intentional pairs). Acceptance criteria for "working" should include precision, not just non-empty output.
- **`refs`** — works under `--global`; quality not separately measured but shares the cross-project ref-detection lineage with `pairs`.
