# bv-era artifacts

Pre-fork project documents from when the project was called `bv` / beads_viewer. Kept as reference for understanding the project's heritage and the original optimization work that informed bt's current performance characteristics.

## Layout

| Path | What |
|---|---|
| `optimization/` | Five-file optimization-round series — two plans (GPT + Opus, ROUND_1) and three analyses (RESULTS_ROUND_1, ANALYSIS_ROUND_2, OPPORTUNITY_MATRIX). Pre-fork performance work. |
| `CLEANED_UP_PROMPTS_USED_TO_CREATE_PROJECT.md` | Genesis prompts used to bootstrap beads_viewer originally. |
| `AGENT_FRIENDLINESS_REPORT.md` | bv-era assessment of the project's agent UX. |
| `UPGRADE_LOG.md` | Pre-fork upgrade history. |
| `GOLANG_BEST_PRACTICES.md` | Generic Go style guide written for bv. References `cmd/bv/main.go`; superseded by AGENTS.md and docs/design/testing.md. |
| `complementary-features-analysis.md` | "What could we add" brainstorm written against a 9-epic bv-era project state. Heavy `bv-N` issue refs. |
| `TOON_INTEGRATION_BRIEF.md` | bd-19e working brief on TOON output integration, RedStone (claude-code/opus-4.5), 2026-01-23. |
| `SKILL.md` | Pre-Cobra Claude Code skill for bt. Uses `--robot-*` flag form and JSONL backend assumptions. **Successor lives in marketplace at `mkt-u8zc`** (paired with bt-u8zc) — adapt content, do not edit this file in place. |

## Period-accuracy norm

These files reference `bv` binary names, `bv-NNN` issue IDs, and pre-Charm-v2 architecture. Don't update them; their period-accuracy is the point. If the *content* is needed in modernized form, copy to a new doc with a current date and current naming, leaving this version intact.

## When something here is still load-bearing

If a file in this folder is being actively cited by current docs, that's a signal: either the citation should point at a modernized successor (preferred) or the file should be promoted out of `bv-era/` (rare). When in doubt, file a bead and ask.
