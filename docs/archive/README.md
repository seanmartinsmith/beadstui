# Archive

Read-only repository of historical artifacts from this project's lifecycle. Not actively maintained. Period accuracy is the value here — don't update content to match current reality.

## Layout

| Folder | What goes here |
|---|---|
| `inbox/` | Holding area for new archives awaiting categorization. Drain to a categorized folder before the session ends or when ≥3 items accumulate. Sort with the user when in doubt. |
| `plans/` | Executed implementation plans. Filed when a plan's work has shipped or when the plan is superseded. |
| `audits/` | Audit reports, scouts, inventories, and baselines. State assessments at a point in time. |
| `bv-era/` | Pre-fork artifacts from when the project was called `bv` / beads_viewer. Period-specific; preserved for reference, not for emulation. See `bv-era/README.md`. |

## Conventions

- **Filenames stay original.** Don't rename to "modernize" content; the filename is part of the historical signal.
- **Internal references inside archived files** (e.g., `docs/plans/...` paths) get updated when files move, but content is otherwise frozen.
- **New archives that don't fit existing buckets** go to `inbox/` rather than creating a one-off folder. Cluster into a real folder only when the bucket has 3+ files.
- **Stale URLs and outdated naming are expected.** This is the only place where `bv-N` issue refs, `steveyegge/beads` URLs, and pre-Charm-v2 architecture descriptions are correct — they reflect the moment the artifact was written.
- **AGENTS.md applies as a fallback.** When in doubt, see the *Docs Structure Conventions* table in `AGENTS.md` at repo root.
